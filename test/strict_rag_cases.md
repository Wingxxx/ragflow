# Strict Citation 多案例测试用例

## 测试用例 1：简单事实查询

### 测试目标
验证 LLM 能正确引用文档中的简单事实

### 测试数据
文档内容：
```
联想智能云教室主要功能：
1. 镜像管理：基于终端样机上传的所有磁盘分区镜像均存储在服务器端数据库中
2. 磁盘管理：依据不同教学需要，灵活配置终端机磁盘分区镜像
3. 分组管理：将所有终端电脑划分为不同分组，便于统一管理
```

### 测试用例
| 用例ID | 问题 | 期望答案(关闭) | 期望答案(开启) |
|--------|------|---------------|---------------|
| TC-001 | 联想云教室的主要功能有哪些？ | 列出三大功能（镜像、磁盘、分组管理） | 同左，但必须引用原文 |
| TC-002 | 磁盘管理功能是什么？ | 说明磁盘管理的作用 | 同左，但必须引用原文 |

## 测试用例 6：多文件综合查询

### 测试目标
验证跨文件引用的准确性

### 测试数据
文件1：
```
2023 年公司收入为 100 万美元。
```
文件2：
```
2023 年公司支出为 80 万美元。
```

### 测试用例
| 用例ID | 问题 | 期望答案(关闭) | 期望答案(开启) |
|--------|------|---------------|---------------|
| TC-011 | 2023 年公司的收入和支出分别是多少？ | 收入 100 万，支出 80 万（可能加推断） | 收入 100 万 [ID:1]，支出 80 万 [ID:2]，必须分别引用 |
| TC-012 | 2023 年公司盈利多少？ | 可能回答 20 万（自行计算） | 根据现有资料无法回答（因为需要计算） |

## 测试执行方法

### 手动测试
1. 启动 RagFlow 服务
2. 上传包含上述测试数据的文档
3. 创建知识库并建立聊天助手
4. 分别在 strict_citation=false 和 =true 模式下测试
5. 对比答案质量和引用准确性

### 自动化测试
```bash
python test/strict_rag_eval.py --mode=comparison
```
